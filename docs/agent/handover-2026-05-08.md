## Four design questions that need answers before writing the convention

These were put to the user at end of session. Answers will determine the content
of `docs/agent/workflows/report-filing.md` and whether the optional code change
is worth scoping.

**Q1 — Per-client vs central filing**
Should new reports land *inside each client's Drive folder*, or stay in one
central folder with per-client filename prefixes?

- Current behaviour: central (`GOOGLE_DRIVE_REPORTS_FOLDER_ID` in `.env`)
- Likely direction: per-client
- If per-client: each client brief already has a `folder_id` — the filing logic
  just needs to resolve `client_name → folder_id → subfolder_id`.

**Q2 — Time bucketing**
Once inside the client folder, should the SEO report subfolder be:
- (a) flat `SEO Reports/` — date already in filename via `dated_title()`
- (b) `SEO Reports/2026/`
- (c) `SEO Reports/2026-05/`

Option (a) is simplest and consistent with current `dated_title()` convention.

**Q3 — Folder auto-creation**
If the target subfolder (e.g. `SEO Reports`) doesn't exist yet for a client:
- Auto-create it silently
- Fail loudly so it gets created manually first

Auto-create = less onboarding friction but can create folder drift.
Fail loud = forces intentional setup, safe for production.

**Q4 — Template enforcement**
`_ CLIENT TEMPLATE` sets the canonical subfolder layout. Should it be applied
retroactively to existing clients, or only enforced for new ones going forward?

**`src/seo_automation_mcp/google_clients.py`**

Replace `_move_to_reports_folder(self, file_id)` with a new method:

```python
def _move_to_folder(self, file_id: str, folder_id: str) -> None:
    """Move *file_id* into *folder_id*, removing all prior parents."""
    drive = self._service("drive", "v3")
    file = drive.files().get(fileId=file_id, fields="parents").execute()
    previous_parents = ",".join(file.get("parents", []))
    kwargs = {
        "fileId": file_id,
        "addParents": folder_id,
        "fields": "id, parents",
    }
    if previous_parents:
        kwargs["removeParents"] = previous_parents
    drive.files().update(**kwargs).execute()

def _move_to_reports_folder(self, file_id: str) -> None:
    """Backwards-compat shim — uses the global fallback folder."""
    self._move_to_folder(file_id, self.settings.require_reports_folder_id())
```

**`src/seo_automation_mcp/workflows.py`**

Each `create_*` workflow currently calls `google.create_sheet(...)` and
`google.create_doc(...)` which internally call `_move_to_reports_folder`.

3. Pass `report_folder_id` into `create_sheet` / `create_doc`, which call
